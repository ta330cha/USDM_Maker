# Issue #001 背景ノードの扱い

## 現状

背景は255件まで登録可能としているが、
画面上でどのように配置・表示するかが未定義。

## 決定

案1を採用する。背景ノードを最上位としてキャンバスに表示し、
各上位要求は必ず1つの背景ノードに属するものとする。

上位要求に複数の背景ノードを親として設定することは認めない。

## 候補

### 案1

背景ノードを最上位として表示する。

```
背景
 ↓
上位要求
```

### 案2

背景ノードを独立して複数表示する。

```
背景1
背景2
背景3

 ↓

上位要求
```

### 案3

背景は画面左上に一覧表示し、
グラフには表示しない。

## 優先度

高

## 状態

決定済み

# Issue #003 仕様の共有

## 現状

仕様は1つの下位要求に属する前提。

しかし

```
要求A
   \
    \
    仕様
    /
要求B
```

というケースが存在する。

## 決定

仕様は必ず1つの下位要求に属するものとする。
複数の下位要求を親として設定することは認めない。

所属先以外の下位要求との関係は、所有関係ではなく任意の関連情報として保存できる。
YAML では `related_requirement_ids` に関連する下位要求 ID を配列で保存する。
この関連情報は仕様の親子関係や ID 採番を変更しない。

```yaml
specifications:
  - id: "001-001-001"
    specification: 編集内容を一定時間ごとに保存する
    parent_requirement_id: "001-001"
    related_requirement_ids:
      - "002-001"
```

## 検討事項

仕様を共有可能にするか。

## 優先度

高

## 状態

決定済み

理由は、上位要求、下位要求、仕様と表記規則を揃えられるため。
