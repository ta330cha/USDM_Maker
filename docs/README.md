# USDM MindMap Editor ドキュメント

作成日: 2026-07-15

## 概要

USDM MindMap Editor は、USDM（Universal Specification Describing Manner）の要求仕様をマインドマップ形式で整理・編集する Web アプリケーションである。

要求、理由、仕様をノードとして追加し、要求から仕様までの関係を視覚的に扱えるようにする。保存形式は YAML とし、Git 管理しやすい人間可読なテキストとして保持する。

## 目的

- 要求整理を視覚的に行えるようにする
- 上位要求、下位要求、仕様の階層関係を明確にする
- 要求から仕様へのトレーサビリティを維持する
- Git で差分確認しやすい YAML 形式で保存する
- ID 採番を自動化し、編集時の手作業を減らす

## 想定アプリケーション

- 種別: ローカル Web アプリケーション
- 技術: ASP.NET Core + Blazor Web App
- 実行環境: macOS / Apple Silicon を主対象
- 初期開発: `dotnet run` によるローカル起動
- 保存形式: YAML ファイル

## 初期スコープ

初期実装では、以下を対象とする。

- キャンバス上でのノード表示
- 背景、上位要求、下位要求、仕様の追加
- 要求、理由、仕様本文の編集
- 単一親の親子関係の管理
- 仕様に対する関連下位要求の記録
- ID 自動採番
- 各階層255件を上限とする ID 管理
- 自動整列・自動配置
- 固定配色と理由の折りたたみ表示
- Undo / Redo (`Cmd+Z` / `Cmd+Y`)
- macOS のコピー＆ペースト (`Cmd+C` / `Cmd+V`)
- 要求、理由、仕様を対象とするノード検索
- 親ノード削除時の配下全削除と削除確認
- YAML のブラウザダウンロード保存
- YAML からの読み込み
- ズーム、パン
- エクスポートメニューからの USDM 表ダウンロード
- エクスポート時の理由未入力警告

以下は未決定事項を整理したうえで段階的に実装する。

- YAML スキーマバージョン

親子関係は単一親とする。上位要求は1つの背景、下位要求は1つの上位要求、仕様は1つの下位要求にのみ属する。仕様に関連する下位要求がある場合は、親子関係にせず関連 ID として保存する。

ノード位置は保存せず、自動整列・自動配置で再現する。接続線はベジェ曲線とし、YAML の保存はブラウザダウンロードとして行う。理由は任意だが、USDM 表などのファイルエクスポート時には未入力を警告する。Google Drive 連携とユーザーによる配色変更は将来対応とする。

## 開発方針

ASP.NET Core + Blazor Web App を前提に、C# 中心で実装する。マインドマップ描画は Blazor コンポーネントと SVG / HTML / CSS を組み合わせて実現する。

データ処理、ID 採番、YAML 入出力、検証処理は Core 層へ分離し、UI 層から独立してテストできる構成とする。
