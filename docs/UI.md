# USDM MindMap Editor UI 設計書

作成日: 2026-07-15

## 1. 目的

本書は、USDM MindMap Editor の画面構成、操作、ノード表示、ダイアログ、状態表示を定義する。

## 2. 基本方針

- マインドマップ感覚で要求と仕様を編集できること
- ノード追加操作を少なくし、入力に集中できること
- ID はユーザーが手入力せず、自動採番すること
- YAML への保存結果が Git 管理しやすいこと
- ノード間の関係が一目で分かること

## 3. 画面構成

```text
┌──────────────────────────────────────────────┐
│ ツールバー                                    │
├──────────────────────────────────────────────┤
│                                              │
│                 キャンバス                    │
│                                              │
│  背景 / 上位要求 / 下位要求 / 仕様ノード       │
│                                              │
├──────────────────────────────────────────────┤
│ ステータスバー                                │
└──────────────────────────────────────────────┘
```

## 4. ツールバー

初期実装で想定する操作:

| 操作 | 内容 |
|---|---|
| 新規 | 空の USDM 文書を作成する |
| 開く | YAML ファイルを読み込む |
| 保存 | YAML ファイルとして保存する |
| 背景追加 | 背景項目を追加する |
| 自動整列 | ノードを自動配置する |
| ズームリセット | キャンバスの拡大率と位置を初期化する |

Undo / Redo は要求として重要だが、詳細仕様が未決定のため `issues.md` の確認事項とする。

## 5. キャンバス

キャンバスは以下を満たす。

- 無限スクロール相当の広い作業領域
- ドラッグによるパン
- ホイールまたは UI 操作によるズーム
- ノードのドラッグ移動
- 接続線の表示
- ノード選択状態の表示

## 6. ノード種別

### 背景ノード

表示項目:

- ID
- 背景本文

背景をキャンバス上に表示するか、一覧として扱うかは未決定。設計上はノード表示と一覧表示の両方に対応できるようにする。

### 上位要求ノード

表示項目:

- ID
- 要求
- 理由

操作:

- 下位要求追加
- 編集
- 削除
- 折り畳み

### 下位要求ノード

表示項目:

- ID
- 要求
- 理由

操作:

- 仕様追加
- 編集
- 削除
- 折り畳み

### 仕様ノード

表示項目:

- ID
- 仕様

操作:

- 編集
- 削除

## 7. ノード配置

初期配置ルール:

- 上位要求はキャンバス中央付近に配置する
- 下位要求は親の下方向に配置する
- 仕様は下位要求の右方向に配置する
- 同一親配下のノードは重ならないようにずらして配置する

接続方向:

```text
背景
 ↓
上位要求
 ↓
下位要求
 →
仕様
```

## 8. ノード追加操作

### 上位要求追加

```text
キャンバスの空白をダブルクリック
  ↓
入力ダイアログ表示
  ↓
要求、理由を入力
  ↓
保存
  ↓
上位要求ノード生成
```

### 下位要求追加

```text
上位要求ノードの追加ボタンを押す
  ↓
入力ダイアログ表示
  ↓
要求、理由を入力
  ↓
保存
  ↓
下位要求ノード生成
```

### 仕様追加

```text
下位要求ノードの追加ボタンを押す
  ↓
入力ダイアログ表示
  ↓
仕様を入力
  ↓
保存
  ↓
仕様ノード生成
```

## 9. 入力ダイアログ

### 要求入力

項目:

- 要求
- 理由

入力ルール:

- 要求は必須
- 理由は初期仕様では必須扱いを推奨する
- ID は表示のみ、または保存後に自動表示する

### 仕様入力

項目:

- 仕様

入力ルール:

- 仕様は必須
- ID は表示のみ、または保存後に自動表示する

### 背景入力

項目:

- 背景

入力ルール:

- 背景は必須

## 10. 理由の表示

理由の表示方法は未決定。初期設計では以下を推奨する。

- ノード内では2行程度まで表示する
- 長文は折り畳む
- ノード選択時または編集ダイアログで全文を確認する

## 11. 接続線

接続線は SVG で描画する。

初期推奨:

- 線種: ベジェ曲線
- 矢印: 親から子へ向かう方向を表示
- 選択中ノードに関係する線を強調表示

最終デザインは `issues.md` の決定に従う。

## 12. ステータスバー

表示候補:

- 現在の YAML ファイル名
- 未保存変更の有無
- ノード数
- ズーム率
- 検証エラー件数

## 13. キーボード操作

初期候補:

| 操作 | キー |
|---|---|
| 保存 | `Cmd+S` |
| Undo | `Cmd+Z` |
| Redo | `Cmd+Shift+Z` |
| 選択ノード削除 | `Delete` |
| ズームリセット | `0` |

ショートカットは macOS 前提のため `Cmd` 系を基本とする。Windows 対応が必要になった場合は `Ctrl` 系も追加する。
