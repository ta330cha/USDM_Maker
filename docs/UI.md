# USDM_MindMap_Editor UI 設計書

作成日: 2026-07-15

## 1. 目的

本書は、USDM_MindMap_Editor の画面構成、操作、ノード表示、ダイアログ、状態表示を定義する。

## 2. 基本方針

- マインドマップ感覚で要求と仕様を編集できること
- ノード追加操作を少なくし、入力に集中できること
- ID はユーザーが手入力せず、自動採番すること
- YAML への保存結果が Git 管理しやすいこと
- ノード間の関係が一目で分かること

## 3. 画面構成

```text
┌──────────────────────────────────────────────┐
│ ツールバー                                    │
├──────────────────────────────────────────────┤
│                                              │
│                 キャンバス                    │
│                                              │
│  背景 / 上位要求 / 下位要求 / 仕様ノード       │
│                                              │
├──────────────────────────────────────────────┤
│ ステータスバー                                │
└──────────────────────────────────────────────┘
```

## 4. ツールバー

初期実装で想定する操作:

| 操作 | 内容 |
|---|---|
| 新規 | 空の USDM 文書を作成する |
| 開く | YAML ファイルを読み込む |
| 保存 | YAML ファイルをブラウザのダウンロードとして保存する |
| 背景追加 | 背景項目を追加する |
| 検索 | 要求、理由、仕様を検索する |
| エクスポート | メニューから USDM 表へ変換してダウンロードする |
| ズームリセット | キャンバスの拡大率と位置を初期化する |

ノード構造が変わった場合と YAML 読み込み後は、自動整列を実行する。手動でノード位置を調整する操作は提供しない。

## 5. キャンバス

キャンバスは以下を満たす。

- 無限スクロール相当の広い作業領域
- ドラッグによるパン
- ホイールまたは UI 操作によるズーム
- 自動整列・自動配置されたノードの表示
- 接続線の表示
- ノード選択状態の表示

## 6. ノード種別

### 背景ノード

表示項目:

- ID
- 背景本文

背景ノードはキャンバス上の最上位に表示する。上位要求は背景ノードの子として表示し、背景ノードの追加ボタンから作成する。

### 上位要求ノード

表示項目:

- ID
- 要求
- 理由

操作:

- 下位要求追加
- 編集
- 削除
- 折り畳み

### 下位要求ノード

表示項目:

- ID
- 要求
- 理由

操作:

- 仕様追加
- 編集
- 削除
- 折り畳み

### 仕様ノード

表示項目:

- ID
- 仕様
- 関連する下位要求数

操作:

- 編集
- 削除
- 関連する下位要求の設定

仕様の関連する下位要求は、所属先とは別の参照としてノード内に件数を表示する。親子接続線は所属先の下位要求との間だけに表示する。

### 配色

初期版の配色は以下とする。

| 対象 | 配色 |
|---|---|
| 背景ノード | 青 |
| 要求ノード | 緑 |
| 仕様ノード | 黄 |
| ノードの文字と枠線 | 黒 |
| キャンバス | 白 |

配色のユーザー設定は将来対応とする。

## 7. ノード配置

初期配置ルール:

- 背景はキャンバス上部に配置する
- 上位要求は親の背景ノードの下方に配置する
- 下位要求は親の下方向に配置する
- 仕様は下位要求の右方向に配置する
- 同一親配下のノードは重ならないようにずらして配置する

接続方向:

```text
背景
 ↓
上位要求
 ↓
下位要求
 →
仕様
```

## 8. ノード追加操作

### 上位要求追加

```text
背景ノードの追加ボタンを押す
  ↓
入力ダイアログ表示
  ↓
要求、理由を入力
  ↓
保存
  ↓
選択した背景を親とする上位要求ノード生成
```

### 下位要求追加

```text
上位要求ノードの追加ボタンを押す
  ↓
入力ダイアログ表示
  ↓
要求、理由を入力
  ↓
保存
  ↓
下位要求ノード生成
```

### 仕様追加

```text
下位要求ノードの追加ボタンを押す
  ↓
入力ダイアログ表示
  ↓
仕様を入力
  ↓
保存
  ↓
仕様ノード生成
```

上位要求、下位要求、仕様の追加時に親を複数選択する操作は提供しない。

## 9. 入力ダイアログ

### 要求入力

項目:

- 要求
- 理由

入力ルール:

- 要求は必須
- 理由は任意
- ID は表示のみ、または保存後に自動表示する

### 仕様入力

項目:

- 仕様
- 関連する下位要求（任意、複数選択）

入力ルール:

- 仕様は必須
- ID は表示のみ、または保存後に自動表示する
- 所属先の下位要求は `parent_requirement_id` として必ず1つ保存する
- 関連する下位要求は `related_requirement_ids` として保存し、所属先と同じ下位要求は選択できない

### 背景入力

項目:

- 背景

入力ルール:

- 背景は必須

## 10. ノード削除

親ノードを削除する場合は、配下の子要素をすべて削除する。削除確認ダイアログには削除対象のノード数を表示する。

確認ダイアログには「今後この確認を表示しない」チェックボックスを置く。選択結果はブラウザに保存し、以降の削除では確認ダイアログを表示しない。設定は将来の設定画面から再有効化できるようにする。

## 11. ノード検索

検索対象は要求、理由、仕様とする。ID は検索対象に含めない。

- 要求検索: 要求本文を対象に検索する
- 理由検索: 理由本文を対象に検索する
- 仕様検索: 仕様本文を対象に検索する
- フリーワード検索: 要求、理由、仕様を横断して検索する

検索結果では一致したノードを一覧表示し、選択するとキャンバス上の該当ノードを強調表示する。

## 12. 理由の表示

理由はデフォルトで折りたたんで表示する。
上位要求と下位要求の右端に V ボタンを配置し、押下時に理由を展開または折りたたむ。
展開中は理由の全文をノード内に表示する。

## 13. エクスポート時の警告

YAML 保存では、理由が空でも警告を表示しない。
USDM 表、Excel などのファイルエクスポート時は、理由が空の要求を警告一覧に表示する。
警告は出力を妨げず、ユーザーは確認後にダウンロードを続行できる。

## 14. 接続線

接続線は SVG で描画する。

- 線種: ベジェ曲線
- 矢印: 親から子へ向かう方向を表示
- 選択中ノードに関係する線を強調表示

ベジェ曲線は親子関係の接続線にだけ使用し、関連する下位要求への参照には接続線を表示しない。

## 15. ステータスバー

表示候補:

- 現在の YAML ファイル名
- 未保存変更の有無
- ノード数
- ズーム率
- 検証エラー件数

## 16. キーボード操作

初期候補:

| 操作 | キー |
|---|---|
| 保存 | `Cmd+S` |
| Undo | `Cmd+Z` |
| Redo | `Cmd+Y` |
| コピー | `Cmd+C` |
| ペースト | `Cmd+V` |
| 選択ノード削除 | `Delete` |
| ズームリセット | `0` |

コピーとペーストは macOS 版でのみ提供する。Windows 版の `Ctrl+C`、`Ctrl+V` は初期版では実装しない。
